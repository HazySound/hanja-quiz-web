# 한자 퀴즈 웹앱 — 설계 / 로드맵

데스크탑 HanjaQuizMaker의 모바일(PWA) 이식. 이 문서는 **앞으로의 구조**를 미리 박아둔 것.
지금 동작하는 것 위에 **단계적으로** 이행한다(한 번에 갈아엎지 않음).

## 핵심 아키텍처 — 범위(Scope) × 모드(Mode) 분리
- **Scope(무엇)** = 공부할 한자 묶음. 급수(전역) + 다음 중 하나:
  `전체 / 챕터(음가 ㄱ~ㅎ) / 부수 / 획수범위 / 개수(처음부터 N·순서지정) / 목록·오답·단어장`
- **Mode(어떻게)** = `암기(플래시카드)` 또는 `문제(퀴즈: 객/단/단어)`
- 어떤 Scope든 두 Mode로 학습 가능. **플래시카드는 독립 메뉴가 아니라 '암기 모드'**.

### Scope 모델(코드)
```
scope = {
  grade, only,                 // 급수(전역, prefs)
  kind: 'all'|'chapter'|'radical'|'strokes'|'count'|'list',
  initial?, radical?, strokesMin?, strokesMax?, listRef?,
  order: 'seen'|'seq(가나다)'|'strokes'|'random',
  count?,                       // 처음부터 N개
}
resolveScope(all, scope) -> 한자[]    // 필터+정렬+slice
```

## 스케줄러 (학습 계획 → 실천)
```
schedule = {
  id, name,
  scope,                        // 위 Scope
  range: { start, end },        // 날짜
  dayPolicy: { weekend, holiday },  // 주말/공휴일 포함·제외 조합
  perDay,                       // 자동(범위÷학습일) 또는 수동
  modes: ['flash','quiz','read'],   // 그날 할 것
  progress: { [hanjaId]: studiedDate },
}
```
- **생성(시험 대비형, 기본)**: 범위 + 시작/끝 + 학습일정책 → 학습가능일 수 → **범위를 1/n 자동 분배** → 날짜별 plan.
- **오늘 분량** = 오늘 plan의 미학습분 + **밀린 것(planned<오늘 & 미학습)** 보충.
- **밀림 처리**: 오늘로 자동 이월 또는 '보충' 버킷, 표시.
- 공휴일: 연도별 KR 공휴일 데이터(초기엔 주말 토글부터). 수동 제외일도.
- 오늘 분량(작은 Scope) → 암기/문제로 바로 학습.

## 메인 화면(목표 구성)
```
[오늘 학습] (스케줄 있으면)  "오늘 N자 → [암기][문제]"   없으면 "학습 계획 만들기"
[빠른 학습]  범위 빠르게(전체/챕터/부수/획수/개수) → [암기][문제]
[단어장] · [내 목록]
[설정]
```
→ 현재의 "랜덤 풀이 / 챕터별 / 플래시카드"를 **"범위 → 암기 or 문제"** 로 통합.

## 로드맵(단계)
1. **Scope 엔진 + 범위 빌더** (급수[done]+챕터+부수+획수+개수+순서) — 퀴즈·플래시카드 공유
2. **플래시카드에 범위 적용** (지금 '전부' → 범위 선택)  ← 자유출제도 같은 빌더
3. **퀴즈도 범위 빌더로 통합**
4. **스케줄러**(모델+오늘분량+달력)
5. **메인 재배치**(범위→암기/문제)
6. 단어장/내목록 화면, 한자 상세 시트, 설정

## 확정된 결정 / 제약
- **뜻·예시 데이터**: 라이브 조회 안 함. 데스크탑에서 **전체 캐싱 → 커밋(자동 적용)**. (브라우저 CORS + 스크레이핑 유지보수 리스크 회피)
- **외부 사전 링크** → 인앱 **'한자 상세 시트'**(훈음 전체+부수+획수+예시단어) 우선, 외부는 보조/생략.
- **레이아웃/스크롤**: iOS 키보드 — `touchmove` preventDefault로 window 스크롤 차단, `.screen-scroll`만 스크롤(`src/App.jsx`, `index.css`).
- **패드(태블릿) 최적화**: 모바일 완성 후. 레터박스 말고 꽉 차게(2열/사이드바), 세로 고정 가능.
